My Blog: Variable Fonts

The topic I chose to write about is Variable Fonts. Using variable fonts allow you to enable many different variations of font to be used in a single file instead of having to use multiple files to adjust the aspects of your font. Using a variable font is beneficial because it allows you to use these different styles and sizes of fonts to be adjusted for better readability on your webpage. While you may want a header and body paragraph to be the same font, you might not want them to be the same size or color. With variable fonts you can adjust this to your liking and even add visual effects to your text. When setting an axis of variation, it is important to remember the size you want your text displayed at. This sets a permissible range for the text to be displayed.

Using the @font-face rule lets you set what you want the text font to be and set the weight of that font as well. You can import fonts from sites such as google fonts and use those fonts by means of the @font-face. For my example I used the Avenir font and coupled that with an animation delay and font color. Using the effects, I was able to display my font to change colors and animate vertically. Setting the animation text flow to infinite allowed the color changing to never stop running.

What I also used in my variable fonts CSS was the @keyframes rule. The keyframes CSS rule defines a texts style and gives it a waypoint for animation. The keyframes rule gives you more control of your animation rather than using the transitions sequence option. The word you give after using the @keyframes rule is the animation you set your text to display. In my code, I used the text-flow animation coupled with a transform on the Y axis which moves my text vertically.

Some struggles I faced when implementing variable fonts was adjusting the speed and smoothness of my animation. I found that I added nth-child rules to ten different colors, but my text was only displaying 2 different colors. The speed was a challenge because I was not sure how the timing worked between changing it for the color and the up and down movement. The smoothness was difficult to figure out, I tried adjusting so that it would transition smoother up and down, but my animation stayed a little too choppy. I thought I would be able to change that using the transform options in my @keyframes but it did not allow me to do so.
